## Description
Implement JWT token issuance, validation, refresh, and revocation. JWT payload: userId, email, departmentId, role, exp (8h configurable). Use jose library. Redis for token blacklist.

## Acceptance Criteria
- Login success -> JWT with userId, email, departmentId, role, exp (8h)
- Expired JWT -> auto-refresh via refresh token
- Logout -> old JWT returns 401
- Tampered JWT -> 401
